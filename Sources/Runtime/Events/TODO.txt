이벤트 발생 처리 과정

1. 이벤트 발생
    - 사용자의 요청 또는 어플리케이션 내 동작으로 인한 이벤트 발생.

2. 이벤트 수집
    - 발생한 이벤트를 수집하여 이벤트 큐에 저장.
    - 이벤트 큐는 이벤트를 순차적으로 처리하기 위한 구조.

3. 이벤트 분류
    - 수집된 이벤트를 유형별로 분류.
    - 예: 사용자 입력 이벤트, 시스템 이벤트 등.

4. 이벤트 핸들러 호출
    - 분류된 이벤트에 해당하는 핸들러를 호출.
    - 핸들러는 이벤트를 처리하는 로직을 포함.

5. 이벤트 처리
    - 핸들러가 이벤트를 처리하고, 필요한 경우 상태를 업데이트.
    - 예: UI 업데이트, 데이터베이스 변경 등.
